# Weather Forecasting & Analysis (Tech Assessment #3)

A Python data-science pipeline over the Kaggle **[Global Weather Repository]
(https://www.kaggle.com/datasets/nelgiriyewithana/global-weather-repository)**
dataset (daily weather for ~200 capital cities). It cleans the data, explores it,
**forecasts a temperature time series** from the `last_updated` field with
scikit-learn, and runs several advanced analyses — then writes a Markdown report
with charts.

---

## Setup

```bash
cd data-science
python -m venv .venv && source .venv/bin/activate   # optional
pip install -r requirements.txt
```

## Run it

**On the real data** — download `GlobalWeatherRepository.csv` from the Kaggle
link above into this folder, then:

```bash
python src/weather_forecasting.py --input "GlobalWeatherRepository.csv"
```

**Demo mode** — no download needed; runs on a synthetic dataset that matches the
real schema so you can see the whole pipeline immediately:

```bash
python src/weather_forecasting.py --demo
```

Useful options:

```bash
--location "Tokyo"     # forecast one city (default: the most data-rich city)
--location global      # forecast the all-cities mean instead
--horizon 30           # forecast 30 days ahead (default 14)
--outdir outputs       # where charts + REPORT.md are written
```

## What you get (in `outputs/`)

> **Note on the committed `outputs/`:** the report and charts checked in here
> were generated in **`--demo` mode on synthetic data** (the report is clearly
> banner-marked as such) so the deliverable is viewable without the download.
> Re-run with `--input "GlobalWeatherRepository.csv"` to regenerate on real data
> — the banner disappears automatically.

- **REPORT.md** — the written report: cleaning summary, EDA findings, the forecast
  model comparison + metrics, anomaly findings, air-quality correlations, and
  spatial/climate patterns. Starts with the PM Accelerator mission.
- **Charts** — temperature distribution, correlation heatmap, warmest locations,
  the forecast (history + backtest + projection), feature importance, anomaly
  timeline, PM2.5 vs temperature, most-polluted locations, temperature-vs-latitude,
  temperature by latitude band, and a station map colored by temperature.

## Method, briefly

- **Cleaning** — parse `last_updated` to datetime, coerce numeric columns, drop
  duplicates and rows without a temperature, median-fill remaining gaps, and clip
  physically implausible temperatures.
- **Forecasting** — aggregate to a daily mean-temperature series, engineer
  lag (1/2/3/7-day), rolling mean/std, and seasonal (day-of-year sin/cos) features,
  then train **Linear Regression, Random Forest, Gradient Boosting, and an
  averaging ensemble**. Models are evaluated on the held-out final 20% of the
  series (a time-based split, never shuffled), scored with MAE / RMSE / R², and
  the best model drives a **recursive multi-step forecast** for the chosen horizon.
- **Advanced analyses** — statistical (z-score) + multivariate (Isolation Forest)
  anomaly detection; PM2.5 vs weather correlations and the most-polluted cities;
  and climate structure (temperature vs latitude, by latitude band, and a map).

## Notes

- **scikit-learn only.** Classical ML (Random Forest / Gradient Boosting / linear
  models + an ensemble) was chosen over heavier time-series libraries (Prophet,
  statsmodels) to keep the dependency footprint small and the install reliable.
- **Forecast target.** Forecasting a *single city* (the default) gives a clean,
  strongly autocorrelated signal. The global mean across both hemispheres largely
  cancels seasonality, so it's available via `--location global` but isn't the
  default.
- **Demo data is synthetic.** It's generated by `src/make_sample_data.py` purely so
  the pipeline can be exercised without the download; any report produced in demo
  mode is clearly banner-marked and the numbers are illustrative, not real.
